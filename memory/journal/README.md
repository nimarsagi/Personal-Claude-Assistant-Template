One subfolder per project/domain (canonical names come from setup, recorded in MEMORY.md; new ones are created on first use). Each holds three folders: `sessions/` — dated entries (YYYY-MM-DD.md), the memory of the work itself; `lessons/` — LESSONS.md, the key rules distilled from that work (Rule/When/Why format); and `skills/` — that domain's own skills, which become active in a given project only when symlinked into that project's `.claude/skills/` (see this folder's CLAUDE.md, "Managing skills"). Write/read routing rules are in CLAUDE.md too.
